# LLM RolePlay v3.2.2 项目分析报告

## 1. 项目概述

LLM RolePlay 是一个基于大语言模型的角色扮演对话系统，专注于提供沉浸式的角色互动体验。该项目集成了先进的分层记忆管理系统，支持 Web UI 和 CLI 两种使用模式，并提供灵活的角色创建和管理功能。

### 核心特性
- 多角色对话支持
- 分层长期记忆管理（日/周/月/年总结）
- **深度回忆功能（支持三种回忆模式）**
- **LLM 总结算法（Qwen3.5-Flash 思考模式）**
- WebSocket 实时流式对话
- 美观的 Web 界面
- CLI 调试模式
- 多模型支持（通义千问系列）
- 图片上传和多模态对话
- 深度思考和联网搜索功能
- 对话统计和分析

## 4. 核心模块详解

### 4.1 分层记忆管理系统 (HierarchicalMemoryDB)

**文件位置**: `vdb_tools/hierarchical_memory_db.py`

这是项目最核心的模块之一，提供了强大的长期记忆管理和深度回忆能力。

#### 架构设计

```
L0: 瞬时工作记忆 (Context Buffer)
    └─> 滑动窗口，保存最近对话，容量受限 (max_context_length=6000)

L1: 日级总结 (Daily Summary)
    └─> 每天的对话压缩总结，保留 14 篇

L2: 周级总结 (Weekly Summary)
    └─> 每周总结，保留 8 篇

L3: 月级总结 (Monthly Summary)
    └─> 每月总结，保留 12 篇

L4: 年级总结 (Yearly Summary)
    └─> 每年总结，保留 100 篇

原始记录库 (Raw Records)
    └─> 所有原始对话，按半年分区存储，永久保存
```

#### 核心功能

1. **嵌入模型管理**
   - 本地优先加载策略
   - 支持离线模式
   - 自动下载和缓存模型

2. **记忆存储与检索**
   - 分层向量存储
   - 时间路由检索策略
   - 语义相似度匹配

3. **深度回忆功能**
   - 支持三种回忆模式
   - 自动意图检测
   - 灵活的深度配置

4. **LLM 总结算法**
   - Qwen3.5-Flash 思考模式
   - 分层级专用 Prompt 模板
   - 失败时回退到非 LLM 算法

5. **数据保留策略**
   - 自动限制各层级总结数量
   - 原始记录永久保存
   - 按时间分区管理

#### 深度回忆功能详解

深度回忆是系统的核心特色功能，让角色能够"回忆"起过去的重要对话。系统提供两种触发深度回忆的方式和三种回忆模式：

**1. 自动意图检测**

系统会自动检测用户输入中的深度回忆意图，通过以下关键词触发：
- "你还记得...吗"
- "仔细回忆"
- "很久以前"
- "上次...说"
- "最初的"
- "咱们第一次"
- "详细想想"

**2. 三种深度回忆模式**

| 回忆模式 | 描述 | 检索范围 | 适用场景 |
|---------|------|---------|---------|
| **关闭 (off)** | 不进行深度回忆，仅使用短期记忆 | 无 | 日常闲聊、简单对话 |
| **正常 (normal)** | 从分层总结库检索 | 日/周/月/年总结库 | 一般回忆需求，平衡精度和性能 |
| **增强 (enhanced)** | 从原始记录库检索 | 最近半年的原始对话记录 | 需要精确回忆细节时 |

**3. 各模式的检索机制**

- **关闭模式**: 不检索任何记忆，仅使用当前上下文缓冲区
- **正常模式**: 使用 `retrieve_with_time_routing()` 函数：
  - 从日/周/月/年四个总结库中进行语义检索
  - 根据时间范围智能选择检索范围
  - 返回最相关的记忆片段
- **增强模式**: 使用 `retrieve_from_raw_records()` 函数：
  - 直接从最近半年的原始对话记录库检索
  - 提供最精确的回忆效果
  - 但检索范围仅限于最近半年

#### LLM 总结算法详解

v3.2.2 版本引入了全新的 LLM 总结算法，大幅提升了总结质量：

**1. 技术选型**
- **模型**: Qwen3.5-Flash
- **模式**: 思考模式 (enable_think=True)
- **思考预算**: thinking_budget=4096

**2. 分层级 Prompt 模板**

系统为四个层级分别设计了专用的 Prompt 模板：

- **日级总结 Prompt** (SUMMARY_PROMPT_DAILY)
  - 突出重要事件和情感变化
  - 保留关键细节和人物关系
  - 字数控制在 200-400 字之间

- **周级总结 Prompt** (SUMMARY_PROMPT_WEEKLY)
  - 提炼本周核心主题和重要发展
  - 保留情感脉络和关键转折点
  - 字数控制在 300-500 字之间

- **月级总结 Prompt** (SUMMARY_PROMPT_MONTHLY)
  - 提炼本月核心事件和成长变化
  - 保留长期记忆和重要关系发展
  - 字数控制在 400-600 字之间

- **年级总结 Prompt** (SUMMARY_PROMPT_YEARLY)
  - 提炼本年核心主题和重大变化
  - 保留人生重要节点和长期发展
  - 字数控制在 500-800 字之间

**3. 回退机制**

为确保系统稳定性，实现了完善的回退策略：
- 优先使用 LLM 生成总结
- 当 LLM 调用失败或返回空结果时，自动回退到原非 LLM 算法
- 非 LLM 算法基于关键词提取，确保在任何情况下都能生成总结

**4. 工作流程**

```
1. 检查 LLM 适配器是否初始化
   ↓
2. 根据目标层级选择对应的 Prompt 模板
   ↓
3. 将待总结的文本填充到 Prompt 中
   ↓
4. 调用 Qwen3.5-Flash（思考模式）生成总结
   ↓
5. 检查生成结果是否有效
   ↓
6. 有效 → 返回 LLM 生成的总结
   ↓
   无效 → 回退到非 LLM 算法
```

#### 关键方法

| 方法名 | 功能描述 |
|--------|----------|
| `add_memory()` | 添加新记忆到缓冲区和原始数据库 |
| `save_summary()` | 保存总结到指定层级的向量数据库 |
| `retrieve_with_depth_mode()` | 基于深度模式的记忆检索 |
| `retrieve_with_time_routing()` | 时间路由检索（正常模式） |
| `retrieve_from_raw_records()` | 原始记录检索（增强模式） |
| `set_depth_recall_mode()` | 设置深度回忆模式 |
| `get_depth_recall_mode()` | 获取当前深度回忆模式 |
| `_generate_summary_from_texts()` | 生成总结（LLM + 回退） |
| `_fallback_summary()` | 非 LLM 回退算法 |
| `compress_to_weekly()` | 压缩到周级总结 |
| `compress_to_monthly()` | 压缩到月级总结 |
| `compress_to_yearly()` | 压缩到年级总结 |

## 5. 数据流与工作流

### 5.1 完整对话流程

```
1. 用户输入消息
   ↓
2. 前端通过 WebSocket 发送到后端
   ↓
3. ChatService 接收消息
   ↓
4. RoleplaySession 处理:
   a. 检测深度回忆意图（自动检测关键词）
   b. 如启用深度回忆，调用 retrieve_with_depth_mode() 检索相关记忆
   c. 组装消息（系统提示词 + 上下文 + 深层记忆）
   d. 调用 LLM 适配器
   ↓
5. LLM 适配器流式生成回复
   ↓
6. 回复实时通过 WebSocket 传回前端
   ↓
7. 对话结束后:
   a. 保存用户输入到记忆
   b. 保存 AI 回复到记忆
   c. 更新统计数据
   ↓
8. 后台监控线程检测压缩条件
   ↓
9. 如满足条件，触发记忆压缩
   ↓
10. 使用 LLM 生成总结（Qwen3.5-Flash 思考模式）
   ↓
11. 保存总结到对应层级
```

### 5.2 角色创建流程

```
1. 用户在 Web UI 填写角色信息
   ↓
2. 前端发送 POST /api/roles 请求
   ↓
3. RoleManager 创建角色
   ↓
4. HierarchicalMemoryManager 初始化记忆数据库
   ↓
5. 保存角色元数据
   ↓
6. 返回创建成功响应
```

### 5.3 深度回忆工作流

```
1. 用户选择深度回忆模式，或输入带有回忆意图的消息
   ↓
2. 系统检测到深度回忆需求
   ↓
3. 根据当前深度回忆模式确定检索策略:
   - off: 不检索
   - normal: 使用 retrieve_with_time_routing()
   - enhanced: 使用 retrieve_from_raw_records()
   ↓
4. 将用户输入向量化
   ↓
5. 在对应库中进行语义检索
   ↓
6. 获取 top_k 个最相关的记忆片段
   ↓
7. 将记忆片段插入到系统提示词中
   ↓
8. AI 根据系统提示词 + 上下文 + 深层记忆生成回复
```

### 5.4 LLM 总结工作流

```
1. 检测到压缩条件（时间边界或容量边界）
   ↓
2. 从源层级获取待总结的文本
   ↓
3. 调用 _generate_summary_from_texts()
   ↓
4. 检查 LLM 适配器是否可用
   ↓
5. 如果可用:
   a. 根据目标层级选择对应的 Prompt 模板
   b. 填充 Prompt
   c. 调用 Qwen3.5-Flash（enable_think=True, thinking_budget=4096）
   d. 收集流式输出
   ↓
6. 如果 LLM 成功且结果有效:
   a. 记录日志
   b. 返回 LLM 生成的总结
   ↓
7. 如果 LLM 失败或结果无效:
   a. 记录警告日志
   b. 调用 _fallback_summary() 使用非 LLM 算法
   ↓
8. 保存总结到目标层级
   ↓
9. 执行保留策略
```

## 8. 总结

LLM RolePlay v3.2.2 是一个功能完整、架构清晰的角色扮演对话系统。其核心优势在于：

1. **先进的分层记忆管理**: 提供长期记忆能力，让角色能够"记住"过去的对话
2. **强大的深度回忆功能**: 支持三种回忆模式（关闭/正常/增强），让角色能够灵活地回忆起不同时期和精度的记忆
3. **高质量的 LLM 总结**: 使用 Qwen3.5-Flash 思考模式，配合分层级专用 Prompt，大幅提升总结质量
4. **灵活的架构设计**: 模块化设计，易于扩展和维护
5. **双模式支持**: Web UI 和 CLI 两种使用方式，满足不同场景需求
6. **丰富的功能**: 多模型支持、多模态对话、深度思考、联网搜索等
7. **现代化界面**: 美观的 Web UI，良好的用户体验

该项目非常适合作为学习大语言模型应用开发的参考，也可以直接用于构建个性化的 AI 角色对话系统。
